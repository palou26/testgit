Cette dernière étape avant le lancement des calculs permet de modifier les tables STATION et MASSEDEAU et de personnaliser les tables de paramètres.

Les tables STATION et MASSEDEAU ne sont pas importées dans l’application, mais copiées dans un dossier spécifique après l’installation et sont administrables avant le lancement d’un calcul via cette interface.

DESCRIPTION DE LA TABLE « STATION »

Elle doit se situer dans le dossier : MON_DOSSIER_S3R\TABLES\STATION.csv

Elle doit impérativement respecter à minima la structuration et les noms de colonnes suivants :

Colonnes

Description

Nota

STATION

Code de la station

Ce code doit être identique à celui importés avec les jeux de données, sinon les données ne seront pas traitées. 

Valeur nulle interdite

DOMAINE

Domaine piscicole de la station

Information indispensable pour le calcul de la température.

Valeurs possibles :

salmonicole / cyprinicole / intermédiaire.

 

S3R transforme également les informations qui commencent par un « s » en salmonicole / c à cyprinicole / i à intermédiaire.

Cela permet de palier les erreurs de frappes éventuelles.

En cas de valeur nulle, la température ne sera pas considérée sur la station

TYPEFR

 TypeFr de la station

Information indispensable pour le calcul de l’état biologique (IBG et IBD).

 

Au début du traitement S3R met en évidence tous les TYPEFR de cette table qui ne sont pas conformes à ceux des grilles IBD et IBG.

En cas de valeur nulle, la biologie ne sera pas considérée sur la station

EUCD

Code de la masse d’eau sur laquelle est rattachée la station

Ce code doit être identique à celui de la table MASSEDEAU.csv, sinon l’association à la masse d’eau ne sera pas possible pour l’extrapolation des états à la masse d’eau.

TYPESTAT

Typologie de la station selon sa masse d’eau de rattachement

Valeurs possibles (MEFM, MEA, MEN).

Cette valeur est renvoyée dans les fichiers de sortie, elle n’intervient pas dans le calcul de l’état écologique pour les MEFM. L’information est prise dans le fichier masse d’eau.

REPRESENTATIVE

Valeurs possibles :

DESCRIPTION DES TABLES DE PARAMETRES

S3R offre à l’utilisateur la possibilité d’utiliser les paramètres définis dans l’arrêté de janvier 2010 et la version révisé 2015, mais également de faire des simulations en utilisant des tables de paramètres personnalisées ou en modifiant les valeurs seuils suite à une évolution de la réglementation.

L’utilisateur doit donc choisir d’utiliser les paramètres et leurs valeurs tels que dans l’arrêté ou d’utiliser ses propres valeurs seuils. Depuis cette interface l’utilisateur peut administrer les valeurs seuils sans toutefois modifier les noms de colonnes des fichiers csv de paramétrage.

NB : des tables correspondant aux 2ème et 3ème cycles sont créées et leurs contenus sont adaptés aux règles de calcul, elles se nomment :

·         XXXX_2c : pour les tables de paramètres du 2ème cycle

·         XXXX_3c : pour les tables de paramètres du 3ème cycle

Toutes les tables de paramètres n’ont pas forcément un paramétrage pour le 2ème ou 3ème cycle. Dans ce cas c’est la table pour défaut (1er cycle) qui est utilisée.

GRILLEIBD, GRILLEIBG, GRILLEIBMR, GRILLEIPR  : tables contenant les valeurs seuils des IBD, IBG, IBMR, IPR.

·         TYPEFR : code du typeFr

·         INFB : borne inférieure comprise de la classe de qualité bleue (très bon)

·         INFV : borne inférieure comprise de la classe de qualité verte (bon)

·         INFJ : borne inférieure comprise de la classe de qualité jaune (moyen)

·         INFO : borne inférieure comprise de la classe de qualité orange (médiocre)

·         INFR : borne inférieure comprise de la classe de qualité rouge (mauvais)

·         GRILLE : indique le typeFR utilisé pour renseigner les valeurs de la grille en raison d’une absence de référence ou d’un type inexistant dans les grilles de l’arrêté. En résumé les valeurs d’un TypeFR de substitution sont utilisées.

·         HER : HER niveau1. Cette colonne n’est pas utilisée par S3R.

LISTECODERQE : table des codes remarques utilisés

·         REMARQUE : code SANDRE du code remarque

·         REMARQUELIB : libellé du code remarque

·         QUANTIFIE : précise si le code remarque correspond à des valeurs quantifiées ou non

PARAMETREBIO : table des paramètres biologiques

·         PARAMETRE : code SANDRE du paramètre biologique

·         PARAMETRELIB : libellé du paramètre

·         PARAGROUP : code SANDRE du groupe de paramètres

·         PARALIBGROUP : nom du groupe de paramètres

·         IDTRI : identifiant de tri des colonnes de résultats

PARAMETREPCH : table des paramètres physico-chimiques

·         PARAMETRE : code SANDRE du paramètre PCH

·         NOM : nom du paramètre

·         ELTQUALITE : élément de qualité auquel appartient le paramètre

·         FRACTION : code SANDRE de la fraction sur laquelle doit être analysé le paramètre pour l’évaluation

·         IDTRI : identifiant de tri des colonnes de résultats

·         SUPB : borne supérieure de la classe de qualité bleue (très bon). Elle sert uniquement à fixer la limite maximale pour l’indice 100 (indice type Seq-eau), car l’état très bon est uniquement borné par sa limite inférieure dans l’arrêté.

·         INFB : borne inférieure comprise de la classe de qualité bleue (très bon)

·         INFV : borne inférieure comprise de la classe de qualité verte (bon)

·         INFJ : borne inférieure comprise de la classe de qualité jaune (moyen)

·         INFO : borne inférieure comprise de la classe de qualité orange (médiocre)

·         INFR : borne inférieure de la classe de qualité rouge (mauvais). Elle sert uniquement à fixer la limite maximale pour l’indice 0 (indice type Seq-eau), car l’état mauvais est uniquement borné avec la limite supérieure de l’état médiocre dans l’arrêté.

PARAMETREPCH_NO3aesn : table des valeurs seuils pour les Nitrates selon la grille AESN

·         PARAMETRE : code SANDRE du paramètre PCH

·         NOM : nom du paramètre

·         CLASSE : classe d’état

·         SEUIL : valeur de la borne inférieure

PARAMETREPCHEXCEPT : table des paramètres physico-chimiques concernés par les exceptions typologiques et valeurs de température selon le domaine piscicole

·         PARAMETRE : code SANDRE du paramètre PCH

·         NOM : nom du paramètre

·         SUPB : borne supérieure de la classe de qualité bleue (très bon). Elle sert uniquement à fixer la limite maximale pour l’indice 100 (indice type Seq-eau), car l’état très bon est uniquement borné par sa limite inférieure dans l’arrêté.

·         INFB : borne inférieure comprise de la classe de qualité bleue (très bon)

·         INFV : borne inférieure comprise de la classe de qualité verte (bon)

·         INFJ : borne inférieure comprise de la classe de qualité jaune (moyen)

·         INFO : borne inférieure comprise de la classe de qualité orange (médiocre)

·         INFR : borne inférieure de la classe de qualité rouge (mauvais). Elle sert uniquement à fixer la limite maximale pour l’indice 0 (indice type Seq-eau), car l’état mauvais est uniquement borné avec la limite supérieure de l’état médiocre dans l’arrêté.

·         DOMAINE : type de domaine piscicole (indispensable pour le calcul de la température)

PARAMETREPOLSPE : table des polluants spécifiques

·         PARAMETRE : code SANDRE du paramètre PCH

·         PARAMETRELIB : nom du paramètre

·         FRACTION : code SANDRE de la fraction sur laquelle doit être analysé le paramètre pour l’évaluation

·         POLLUANT : nom du groupe de polluants auquel appartient le paramètre

·         NQEMA : valeur de la NQEMA

·         NQEMA2 : valeur de la NQEMA pour le zinc selon la dureté

·         IDTRI : identifiant de tri des colonnes de résultats

·         AESN : liste des polluants retenus par AESN à partir du 2nd cycle (valeur possible oui/non)

·         XXXX: liste des polluants retenus par XXX  à partir du 2nd cycle (valeur possible oui/non)

Nota : XXXX correspond à l’abréviation des autres agences ou offices de l’eau. Il y a une colonne par district.

PARAGROUPCHIM : table des substances chimiques

·         PARAGROUP : code de la substance (code DCE de l’arrêté)

·         PARAGROUPLIB : nom de la substance

·         PARAGROUPLIBCOURT : nom court de la substance

·         GROUPE : substance composée d’un groupe de paramètres chimiques

·         FAMILLE : famille de la substance

·         NQEMA : valeur de la NQEMA

·         NQEMA2 : valeur de la NQEMA du cadmium selon la classe 2 de dureté

·         NQEMA3 : valeur de la NQEMA du cadmium selon la classe 3 de dureté

·         NQEMA4 : valeur de la NQEMA du cadmium selon la classe 4 de dureté

·         NQEMA5 : valeur de la NQEMA du cadmium selon la classe 5 de dureté

·         NQECMA : valeur de la NQECMA

·         NQECMA2 : valeur de la NQECMA du cadmium selon la classe 2 de dureté

·         NQECMA3 : valeur de la NQECMA du cadmium selon la classe 3 de dureté

·         NQECMA4 : valeur de la NQECMA du cadmium selon la classe 4 de dureté

·         NQECMA5 : valeur de la NQECMA du cadmium selon la classe 5 de dureté

·         IDTRI : identifiant de tri des colonnes de résultats

PARAMETRECHIM : table des paramètres chimiques associés aux substances

·         PARAMETRE : code sandre du paramètre

·         PARAMETRELIB : nom du paramètre

·         FRACTION : code SANDRE de la fraction sur laquelle doit être analysé le paramètre pour l’évaluation

·         PRIORITE : niveau de priorité du paramètre appartenant à une substance pour le calcul de la moyenne. Pour les paramètres en groupe le calcul de la moyenne annuelle est fait en priorité en sommant chaque valeur de paramètre composant la substance plutôt qu’en prenant la valeur de la substance si elle est également disponible.

·         PARAGROUP1 : code de la substance à laquelle est rattaché le paramètre

·         PARAGROUP2 : code de la substance 2 à laquelle est rattaché le paramètre (uniquement pour DDT44). Il est considéré une fois seul et une fois en groupe.

PARAMETREUNITE : table des unités par paramètre

·         PARAMETRE : code SANDRE du paramètre

·         NOM : nom du paramètre

·         UNITE : code SANDRE de l’unité du paramètre sur laquelle sont définies les valeurs seuils

·         UNITELIB : nom de l’unité

·         ETAT : mentionne l’état pour lequel est utilisé le paramètre. Il n’est pas utilisé par S3R, mais peut servir uniquement à l’utilisateur pour identifier un paramètre plus rapidement parmi cette longue liste.
