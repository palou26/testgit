Cette interface permet d’importer les données brutes selon les options précédemment choisies.

Les données doivent être sous forme tabulaire dans un fichier CSV, avec le séparateur point-virgule « ; ».

Si les données sont au format Excel ou Access, veillez à les « enregistrer sous » au format CSV.

Les données physico-chimiques et chimiques doivent être contenues dans un seul et même fichier d’import.

Les données biologiques doivent être dans un fichier spécifique à la biologie. Il est indispensable d’intégrer les IPR Onema à ce fichier, s’ils doivent être pris en compte.

Si des données PCH modélisées sont intégrées, elles doivent être dans un fichier spécifique.

 

DONNEES PHYSICO-CHIMIQUES & CHIMIQUES

A minima le fichier doit contenir les colonnes suivantes :

·         Code station

·         Code paramètre (format SANDRE)

·         Date de prélèvement

·         Heure de prélèvement (possibilité d’omettre cette colonne si elle n’est pas disponible et/ou de l’utiliser pour un autre champ faisant office de clé multiple (code réseau par exemple)).

·         Code fraction (format SANDRE)

·         Résultat brut du paramètre

·         Code remarque (format SANDRE)

·         Code de l’unité (format SANDRE)

·         Limite de quantification

L’étape suivante permet d’associer chaque colonne du fichier importé aux colonnes attendues. Il n’est donc pas nécessaire de respecter l’ordre des colonnes et leur nom. Le format détaillé du fichier est spécifié dans l’aide des fenêtres suivantes de S3R

DONNEES BIOLOGIQUES

A minima le fichier doit contenir les colonnes suivantes :

·         Code station

·         Code paramètre (format SANDRE)

·         Date de prélèvement

·         Heure de prélèvement (possibilité d’omettre cette colonne si elle n’est pas disponible)

·         Résultat brut du paramètre

L’étape suivante permet d’associer chaque colonne du fichier importé aux colonnes attendues. Il n’est donc pas nécessaire de respecter l’ordre des colonnes et leur nom. Le format détaillé du fichier est spécifié dans l’aide des fenêtres suivantes de S3R

DONNEES MODELISEES

A minima le fichier doit contenir les colonnes suivantes :

·         Code station

·         O2 : résultat modélisé

·         Taux de saturation O2 : résultat modélisé

·         DBO5 : résultat modélisé

·         COD : résultat modélisé

·         O2 : résultat modélisé

·         PO43 : résultat modélisé

·         Phosphore total : résultat modélisé

·         NH4 : résultat modélisé

·         NO2 : résultat modélisé

·         NO3 : résultat modélisé

·         Température : résultat modélisé (possibilité d’omettre ce paramètre)

·         pH min : résultat modélisé (possibilité d’omettre ce paramètre)

·         pH max : résultat modélisé (possibilité d’omettre ce paramètre)

Si le fichier ne contient qu’une seule colonne « pH », il faudra l’associer aux champs phmin et phmax.

L’utilisateur doit également préciser si les données contenues sont les valeurs brutes modélisées ou les classes d’état.

L’étape suivante permet d’associer chaque colonne du fichier importé aux colonnes attendues. Il n’est donc pas nécessaire de respecter l’ordre des colonnes et leur nom.

TABLE STATION

L’utilisateur doit indiquer la table « station.csv » qui sera utilisée pour déterminer les calculs. Il s’agit de la table de référence contenant les informations indispensables aux calculs (domaine piscicole, type FR, exception typologique…).

Les champs obligatoires pour cette table sont présentés ultérieurement.

TABLE MASSE D’EAU

L’utilisateur doit indiquer la table « massedeau.csv » qui sera utilisée pour déterminer les calculs. Il s’agit de la table de référence contenant les informations indispensables aux calculs (statut…).

Les champs obligatoires pour cette table sont présentés ultérieurement.
