Cette interface permet d’associer les colonnes du fichier importé pour les données physico-chimiques et chimiques aux colonnes attendues pour le calcul des états.

Nom de colonne

Description

Nota

STATION

Colonne contenant le code station

Il faut s’assurer que le code station est répertorié dans la table « station.csv ».

Avant d’exécuter les traitements S3R signale les incohérences.

DATEPRELEV

Colonne des dates de prélèvement

Aucun filtre n’est fait sur la date de prélèvement. L’utilisateur doit veiller à n’importer que des données sur la période qu’il souhaite évaluer.

HEUREPRELEV

Colonne des heures de prélèvement

L’utilisateur peut omettre cette colonne (option « non_dispo ») si l’heure est associée à la date dans le champ date de prélèvement, ou si elle n’est pas disponible. Si elle n’est pas disponible, c’est un risque de doublon potentiel.

PARAMETRE

Colonne des codes paramètres

Format SANDRE indispensable

Les paramètres non utiles à l’évaluation ne seront pas traités (MES, Azote Kejdal…), il n’est donc pas nécessaire de limiter les données aux seuls paramètres à traiter.

FRACTION

Colonne des codes fraction

Format SANDRE indispensable

S3R gère les différentes codifications des codes fraction (ex : 003 ou 03à 3 ou 023 à 23).

Seules les données sur support « eau » sont traitées. Les sédiments ne sont donc pas pris en compte.

RESULTAT

Colonne des résultats des valeurs de paramètres.

S3R gère la décimale « . » et  « ,»

UNITE

Colonne des unités

Format SANDRE indispensable

L’utilisateur peut omettre cette colonne (option « non_dispo ») si l’unité n’est pas disponible. L’utilisateur a conscience qu’il part du principe que les unités sont correctes. S3R ne fait pas de conversion d’unité si elles ne sont pas conformes au fichier « PARAMETREUNITE.csv ».

REMARQUE

Colonne des codes remarque

Format SANDRE indispensable

S3R gère le cas du code remarque qui est contenu dans le champ résultat (ex : <2.5, sera transformé en résultat = 2.5 et code remarque 10). Toutefois il ne met pas de code remarque pour ceux n’étant pas renseignés. Ils ne seront dons pas traités.

LQ

Colonne des limites de quantification

Si la donnée est présente, elle permet le calcul des LQMAX. Si la donnée n’est pas présente la LQMAX sera déterminée à partir des valeurs des prélèvements non quantifiés. Cette LQ est également utilisée pour remplacer les valeurs dans le calcul des moyennes..

 

S3R va créer « temporairement » une nouvelle table avec de nouveaux noms de colonne.

Pour des évaluations futures et/ou des simulations en fonction des options choisies, il est possible d’enregistrer cette association de colonnes.
